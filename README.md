# Calc for Obsidian

Плагин для Obsidian, который превращает Markdown-блоки в небольшие
интерактивные ячейки:

- считает выражения в блоках `calc`;
- запускает Python-код прямо в заметке через `python run`;
- показывает ошибки Ruff для Python-ячеек;
- вставляет fenced code blocks через компактное меню с историей выбора.

Плагин предназначен для desktop-версии Obsidian.

## Установка

### Вручную

1. Скачайте `main.js`, `manifest.json` и `styles.css`.
2. Создайте папку:

   ```text
   <vault>/.obsidian/plugins/obsidian-calc/
   ```

3. Поместите файлы плагина в эту папку.
4. В Obsidian откройте `Settings -> Community plugins`, включите `Calc`.

При обновлении замените те же три файла и перезагрузите плагин.

## Калькулятор

Создайте блок:

````markdown
```calc
5 + 3
```
````

В Live Preview блок отображается как результат:

```text
8
```

Клик по результату открывает исходное выражение для редактирования.

### Сохранить результат

Внутри `calc` допишите `=` и нажмите `Enter`:

````markdown
```calc
sin(30) + 2 =
```
````

Выражение заменится вычисленным значением.

### Поддерживаемые операции

```text
+  -  *  /  %  ^  ( )
```

Константы:

```text
pi, e
```

Функции:

```text
sin cos tan asin acos atan
sqrt cbrt ln log lg exp
abs round floor ceil trunc fact
```

Тригонометрические функции используют градусы: `sin(30)` возвращает `0.5`.

## Python-ячейки

Исполняемая Python-ячейка записывается с пробелом в имени языка:

````markdown
```python run
numbers = [2, 3, 5]
print(sum(numbers))
```
````

После выхода из блока в Live Preview плагин выполняет код и показывает вывод
ниже ячейки. При наличии ошибок или предупреждений Ruff они отображаются
отдельной панелью. Если проблем нет, панель линтера не показывается.

Обычные блоки `python` остаются блоками кода и не должны использоваться для
запуска:

````markdown
```python
print("This is just code")
```
````

### Важно о безопасности

`python run` выполняет код на вашем компьютере. Не запускайте такие ячейки из
заметок, которым вы не доверяете.

## Локальный Python и Ruff

По умолчанию настройка `Python command` равна `local`: плагин сначала ищет
виртуальное окружение `.venv` внутри своей папки, а затем использует доступный
системный Python как запасной вариант.

### Windows

Откройте терминал в папке плагина:

```powershell
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install ruff
```

### Linux / macOS

```bash
python3 -m venv .venv
./.venv/bin/python -m pip install ruff
```

Папка `.venv` является локальной и не должна добавляться в репозиторий.

## Горячие клавиши

| Сочетание | Действие |
| --- | --- |
| `Ctrl+K` / `Cmd+K` | Открыть меню вставки code block |
| `Ctrl+Enter` / `Cmd+Enter` | Выйти из текущего fenced code block; для `python run` это запускает обновленную ячейку |
| `Tab` | Принять выбранную Python-подсказку в `python run` |

`Ctrl+Enter` обрабатывается внутри code block самим плагином. Вне code block
сочетание остается доступным другим командам Obsidian и другим плагинам.

## Меню блоков кода

Нажмите `Ctrl+K` / `Cmd+K`, чтобы выбрать тип блока. В меню доступны:

```text
calc, text, python, python run, bash, sql, javascript, typescript, json,
html, css, markdown, yaml, powershell, c, cpp, java, rust, go, latex
```

Последние выбранные элементы автоматически поднимаются наверх списка.

## Настройки

В `Settings -> Calc` доступны:

- `Insert code block hotkey` - включить или отключить прямой вызов меню через
  `Ctrl+K` / `Cmd+K`;
- `Language history` - очистить историю выбора языков;
- `Python command` - выбрать `local`, `auto`, `python3`, `python`, `py` или
  указать полный путь к интерпретатору;
- `Python timeout` - ограничить время выполнения Python-ячейки от 1 до 60
  секунд.

## Файлы плагина

Для распространения нужны:

```text
main.js

```
